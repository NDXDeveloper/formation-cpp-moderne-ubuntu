# 2.1.3 — État 2026 : GCC 15 et Clang 20 — Nouveautés et support C++26 🔥

> **Chapitre 2 · Section 2.1 · Installation des compilateurs** · Niveau Débutant

---

## Introduction

Le paysage des compilateurs C++ évolue à un rythme soutenu. En mars 2026, les versions stables que vous installerez sur Ubuntu sont **GCC 15** (branche 15.x, première release stable en avril 2025) et **Clang 20** (LLVM 20.1, publié fin mars 2025), avec **Clang 21** (LLVM 21.1, publié à l'automne 2025) déjà disponible via le dépôt officiel LLVM.

Parallèlement, le **standard C++26** vient de franchir ses étapes finales : le comité ISO a gelé les fonctionnalités en juin 2025 à Sofia, résolu les commentaires des votes nationaux à Kona (novembre 2025), et la réunion de Londres (23–28 mars 2026) doit finaliser techniquement le draft pour l'envoyer au vote d'approbation définitif. La publication formelle est attendue fin 2026.

Cette section fait le point sur ce que ces compilateurs apportent concrètement à votre workflow de développement aujourd'hui.

---

## GCC 15 : le point complet

### Calendrier de releases

GCC 15.1, la première version stable de la branche 15, a été publiée fin avril 2025. GCC 15.2 a suivi en août 2025, avec plus de 123 corrections de bugs. Le développement de GCC 16 se poursuit en parallèle sur la branche `main`.

### Changement de défaut pour le C

GCC 15 change le langage par défaut pour la compilation C de `-std=gnu17` à `-std=gnu23`. C'est un changement important si vous maintenez du code C legacy : certains idiomes valides en C17 ne le sont plus en C23. Pour le C++, en revanche, le dialecte par défaut reste `-std=gnu++17` dans GCC 15 ; le passage à C++20 par défaut est prévu pour GCC 16.

### Fonctionnalités C++23 et C++26 dans le langage

Les fonctionnalités C++26 implémentées dans GCC 15 comprennent notamment le pack indexing, les attributs sur les structured bindings, le support amélioré des fonctions dont la définition est `= delete`, et le `constexpr` placement new.

Parmi les points concrets que vous pouvez utiliser dès aujourd'hui avec `-std=c++26` (ou `-std=c++2c`) :

**Pack indexing (P2662)** — permet d'accéder directement à un élément d'un parameter pack par son index, sans les contorsions de templates récursifs qui étaient nécessaires auparavant :

```cpp
template <typename... Ts>  
using third_type = Ts...[2];  // Accès direct au 3e type du pack  
```

**Structured bindings dans les conditions (P0963)** — GCC 15 implémente la possibilité de déclarer des structured bindings dans les conditions `if`, `while`, `for` et `switch` :

```cpp
struct Result { int code; std::string message;
    explicit operator bool() const { return code == 0; }
};

if (auto [code, msg] = get_result(); code == 0) {
    std::print("Succès : {}\n", msg);
}
```

**Constexpr placement new (P2747)** — la gestion dynamique de la mémoire dans un contexte `constexpr` s'étend au placement new, ouvrant la porte à davantage d'évaluations à la compilation.

**Delete avec raison (P2573)** — les fonctions `= delete` peuvent désormais porter un message explicatif, améliorant la qualité des diagnostics.

### Améliorations de libstdc++

La bibliothèque standard de GCC (libstdc++) a élargi son support pour les fonctionnalités C++ modernes, incluant `std::expected`, `std::flat_map`, et des en-têtes modulaires supplémentaires alignés sur les derniers drafts ISO.

Le support expérimental de C++26 dans libstdc++ inclut `views::concat`, `views::to_input` et `views::cache_latest`. Les algorithmes de tri et les algorithmes de mémoire brute sont désormais `constexpr`, ce qui permet de les utiliser pendant l'évaluation à la compilation.

### Modules C++20

Les modules C++ ont été considérablement améliorés dans GCC 15, avec notamment des gains de vitesse de compilation grâce à l'amélioration du hachage des spécialisations de templates. Le support reste expérimental et nécessite le flag `-fmodules`, mais GCC 15 marque une avancée significative vers l'utilisabilité pratique des modules.

### Diagnostics et ergonomie

GCC 15 apporte des améliorations notables à la qualité des messages d'erreur :

La possibilité d'émettre des diagnostics dans plusieurs formats simultanément via la nouvelle option `-fdiagnostics-add-output=` est particulièrement utile pour les pipelines CI qui veulent à la fois les messages texte classiques sur stderr et une sortie **SARIF** (Static Analysis Results Interchange Format) dans un fichier, exploitable par les outils d'intégration continue.

Un mode expérimental de présentation améliorée des erreurs de concepts est disponible via l'option `-fdiagnostics-set-output=text:experimental-nesting=yes`, inspiré des travaux de Sy Brand sur les messages d'erreur de concepts adaptés aux humains.

Les diagnostics utilisent désormais la **couleur** pour distinguer visuellement les éléments contrastés (par exemple, dans une erreur de type, le type attendu et le type fourni apparaissent dans des couleurs distinctes).

### Optimisation et performances de compilation

GCC 15 introduit le LTO incrémental via le flag `-flto-incremental`, réduisant significativement les temps de recompilation pour les petites modifications de code — une amélioration majeure pour les développeurs travaillant sur de gros projets avec LTO activé.

Le modèle de coût du vectoriseur par défaut à `-O2` a été amélioré pour gérer les boucles avec un nombre d'itérations inconnu. Le vectoriseur supporte désormais la vectorisation de boucles avec des sorties anticipées (early exits), ce qui élargit le spectre de code automatiquement vectorisé.

### Nouvelles cibles architecturales

Côté ARM, GCC 15 introduit un support approfondi pour les jeux d'instructions avancés comme SME2.2 (Scalable Matrix Extensions), SVE2.2 (Scalable Vector Extensions) et LS64 pour les opérations atomiques — des capacités essentielles pour le calcul scientifique et l'IA. Le support RISC-V continue également de progresser rapidement, en particulier autour de la vectorisation.

## Clang 20 et 21 : le point complet

### Calendrier de releases

**LLVM 20.1** (Clang 20) a été publié fin mars 2025 — c'est la version que vous obtiendrez en installant `clang-20` via le script `llvm.sh` décrit en section 2.1. **LLVM 21.1** (Clang 21) est sorti à l'automne 2025 et est disponible via le dépôt APT officiel LLVM. En mars 2026, LLVM 21 est en version de maintenance (21.1.x) et LLVM 22 est en développement.

### Fonctionnalités C++26

Clang 20.1 étend son support pour C++26 avec des améliorations des structured bindings, du pack indexing et divers ajustements de qualité de vie.

Clang 21 va plus loin, avec notamment :

- Des builtins pour le support de la **trivial relocatability** C++26 (`__builtin_is_cpp_trivially_relocatable`, `__builtin_is_replaceable`, `__builtin_trivially_relocate`) — une fonctionnalité qui permet d'optimiser le déplacement de certains types en mémoire.
- Une nouvelle option `--precompile-reduced-bmi` permettant aux systèmes de build de générer un BMI (Binary Module Interface) réduit pour les unités de modules C++20 importables, facilitant l'intégration des modules dans les pipelines de build.
- Un nouvel avertissement `-Wlifetime-safety` pour activer une analyse intra-procédurale de sécurité des durées de vie, détectant les use-after-free et les bugs de sécurité temporelle — un pas concret vers la sécurité mémoire au moment de la compilation.

### libc++ : bibliothèque standard LLVM

Le hardening de libc++ supporte désormais des sémantiques d'assertions modelées sur les Contrats C++26 : ignore, observe, quick-enforce et enforce. Cette fonctionnalité permet de personnaliser la gestion des violations d'assertions de hardening, facilitant l'adoption du hardening en production.

Pour rappel, sur Ubuntu, Clang utilise **libstdc++** (GNU) par défaut et non libc++. Les avancées de libc++ sont néanmoins importantes car elles préfigurent les idiomes qui finiront par se retrouver dans l'écosystème global.

### Améliorations de clang-tidy et de l'outillage

Les outils extra de Clang 20 et 21 incluent de nouveaux checks pour `clang-tidy`, notamment la détection de patterns liés au C++26, des améliorations de `clang-format` et une meilleure intégration SARIF pour les diagnostics.

### Performance de compilation

LLVM 21 est désormais construit avec l'optimisation PGO (Profile-Guided Optimization) dans certaines distributions comme Fedora, ce qui rend Clang notablement plus rapide pour la compilation de fichiers C et C++.

## C++26 : les grandes fonctionnalités à connaître

Le draft C++26 est « feature complete » et a été envoyé pour le vote de commentaire international. Après la résolution de commentaires à la réunion de Kona en novembre 2025, le comité devrait finaliser techniquement le draft et l'envoyer pour le vote d'approbation final lors de la réunion de Londres du 23 au 28 mars 2026.

Voici les trois fonctionnalités majeures qui définissent ce standard :

### Réflexion statique (Static Reflection)

La réflexion statique permet l'introspection au moment de la compilation sur les types et le comportement, étendant massivement les capacités de métaprogrammation du C++. Herb Sutter, président du comité ISO C++, a décrit cette fonctionnalité comme étant plus transformatrice que n'importe quelle combinaison de dix autres fonctionnalités majeures jamais ajoutées au standard.

Exemple canonique — convertir un enum en chaîne de caractères à la compilation :

```cpp
// C++26 — Réflexion statique
enum Color { red, green, blue };

template <typename E>
    requires std::is_enum_v<E>
constexpr std::string enum_to_string(E value) {
    template for (constexpr auto e : std::meta::members_of(^E)) {
        if (value == [:e:])
            return std::string(std::meta::name_of(e));
    }
    return "<unnamed>";
}

static_assert(enum_to_string(Color::red) == "red");
```

Ce code, qui nécessitait auparavant des macros, des bibliothèques tierces ou de la génération de code externe, devient une simple fonction `constexpr` standard.

> 💡 **État du support :** en mars 2026, la réflexion est en cours d'implémentation dans GCC (avec des premiers patches publics) et dans Clang. Des branches expérimentales sont disponibles sur Compiler Explorer pour tester la syntaxe.

### Contrats (Contracts)

Un contrat spécifie les interfaces des composants logiciels de manière précise et vérifiable. Ces composants sont des fonctions qui remplissent des préconditions, des postconditions et des invariants.

```cpp
// C++26 — Contrats
int divide(int a, int b)
    pre(b != 0)            // Précondition
    post(r: r * b == a)    // Postcondition
{
    return a / b;
}
```

Les contrats offrent quatre sémantiques d'évaluation — **ignore**, **observe**, **enforce** et **quick-enforce** — permettant de contrôler finement le comportement en cas de violation (ignorer, journaliser, terminer proprement ou terminer immédiatement). Cette flexibilité permet d'utiliser les contrats en développement comme en production.

> 💡 **État du support :** GCC et Clang travaillent activement sur l'implémentation. Celle-ci est encore expérimentale dans les deux compilateurs en mars 2026. Voir la section 12.14.1 pour une couverture complète.

### std::execution (Senders/Receivers)

Le framework d'exécution asynchrone basé sur le modèle Senders/Receivers fournit une programmation asynchrone structurée, remplaçant à terme `std::async` et `std::future` par un modèle composable et sans surcoût.

```cpp
// C++26 — std::execution (schématique)
auto work = ex::schedule(scheduler)
          | ex::then([] { return fetch_data(); })
          | ex::then([](Data d) { return process(d); });

ex::sync_wait(std::move(work));
```

Ce modèle est composable, compatible avec le zero-overhead abstraction de C++, et conçu pour s'intégrer avec les schedulers et les thread pools.

> 💡 **État du support :** l'implémentation progresse dans libc++ (LLVM) et libstdc++ (GCC). Des bibliothèques tierces comme **stdexec** (anciennement libunifex) permettent déjà d'expérimenter avec le modèle Senders/Receivers.

### Autres fonctionnalités C++26 notables

Au-delà des « big three », C++26 apporte de nombreuses améliorations :

- **Trivial relocatability** — optimisation du déplacement d'objets en mémoire pour les types « trivialement relocalisables ».
- **Types SIMD data-parallel** (`std::simd`) — abstraction portable des opérations SIMD dans la bibliothèque standard.
- **Ranges parallèles** — combinaison des ranges C++20 avec les politiques d'exécution parallèle.
- **Conteneurs et adaptateurs `constexpr`** — élargissement de ce qui peut être évalué à la compilation.
- **Améliorations de sécurité** — nouvelles règles autour du `delete` sur types incomplets, meilleure détection des comportements indéfinis.

## Quel standard cibler en mars 2026 ?

### C++23 : le choix de production

En mars 2026, **C++23 est le standard le plus récent pleinement exploitable** pour du code de production. Les deux compilateurs majeurs le couvrent intégralement côté langage, et les bibliothèques standard (libstdc++ et libc++) implémentent la grande majorité des ajouts de bibliothèque.

Cela signifie que vous pouvez utiliser en confiance :

- `std::print` et `std::format` pour l'affichage et le formatage ;
- `std::expected` pour la gestion d'erreurs sans exceptions ;
- `std::flat_map` et `std::flat_set` pour des conteneurs cache-friendly ;
- `std::mdspan` pour les vues multidimensionnelles ;
- `std::generator` pour les coroutines génératrices ;
- Deducing `this` pour simplifier le code des classes.

### C++26 : exploration et préparation

C++26 est techniquement en cours de finalisation, mais son **support compilateur est encore partiel**. Les fonctionnalités « petites » (pack indexing, structured bindings dans les conditions, constexpr placement new) sont déjà utilisables. Les fonctionnalités « grandes » (réflexion, contrats, std::execution) sont en cours d'implémentation et ne doivent être utilisées que dans des contextes exploratoires.

La stratégie recommandée :

1. **Compilez en `-std=c++23`** pour votre code de production.
2. **Testez en `-std=c++2c`** dans un répertoire expérimental pour vous familiariser avec les nouvelles fonctionnalités dès qu'elles deviennent disponibles.
3. **Surveillez les release notes** de GCC et LLVM pour identifier le moment où les fonctionnalités C++26 que vous visez atteignent un niveau de maturité suffisant.

### C++20 : toujours pertinent

Si vous travaillez sur un projet qui doit supporter des compilateurs plus anciens (GCC 12/13, Clang 16/17), C++20 reste un excellent choix. Il apporte Concepts, Ranges, Coroutines, `std::span`, `std::jthread`, et l'opérateur spaceship `<=>` — un ensemble de fonctionnalités déjà très riche.

## Suivre l'évolution du support

Quatre ressources de référence pour rester à jour :

**cppreference.com — Compiler support** — la page [C++ compiler support](https://en.cppreference.com/w/cpp/compiler_support) liste chaque fonctionnalité du standard avec la version minimale de GCC, Clang et MSVC qui la supporte. C'est la référence la plus complète et la plus à jour.

**GCC — C++ Standards Support** — la page [gcc.gnu.org/projects/cxx-status.html](https://gcc.gnu.org/projects/cxx-status.html) détaille le support de chaque proposal par version de GCC.

**Clang — C++ Status** — la page [clang.llvm.org/cxx_status.html](https://clang.llvm.org/cxx_status.html) fait de même pour Clang.

**Compiler Explorer (godbolt.org)** — permet de tester instantanément si un extrait de code compile avec une version donnée de GCC ou Clang, directement dans le navigateur. C'est l'outil le plus rapide pour vérifier le support d'une fonctionnalité spécifique.

## Impact sur cette formation

Tout au long de cette formation, nous utiliserons principalement **C++23** comme standard cible, avec des incursions en **C++20** quand une compatibilité plus large est nécessaire. Les fonctionnalités C++26 seront présentées dans les sections dédiées (chapitre 12.14) avec les avertissements appropriés sur leur niveau de maturité.

Les commandes de compilation utilisées dans les exemples seront typiquement :

```bash
# Standard recommandé pour la formation
g++-15 -std=c++23 -Wall -Wextra -Wpedantic -O2 -o programme programme.cpp  
clang++-20 -std=c++23 -Wall -Wextra -Wpedantic -O2 -o programme programme.cpp  
```

Et pour les sections expérimentales C++26 :

```bash
# Exploration C++26
g++-15 -std=c++2c -Wall -Wextra -o programme programme.cpp  
clang++-20 -std=c++2c -Wall -Wextra -o programme programme.cpp  
```

## Ce qu'il faut retenir

- **GCC 15** (stable, version 15.2+) apporte un support C++23 quasi complet, des fonctionnalités C++26 initiales (pack indexing, structured bindings dans les conditions, constexpr placement new), un LTO incrémental, des diagnostics en couleur et en SARIF, et des modules C++20 considérablement améliorés.
- **Clang 20/21** (stable, LLVM 20.1 et 21.1) offre un support C++23 complet, des fonctionnalités C++26 comparables à GCC 15, de nouveaux avertissements de sécurité mémoire (`-Wlifetime-safety`), et un outillage toujours à la pointe (clangd, clang-tidy, clang-format).
- **C++26** est en cours de finalisation par l'ISO (vote final prévu mars 2026, publication fin 2026). Ses trois fonctionnalités majeures — **réflexion statique**, **contrats** et **std::execution** — sont décrites comme les plus transformatrices depuis C++11, mais leur support compilateur est encore partiel.
- **En pratique** : ciblez C++23 pour la production, expérimentez avec C++26 dans des projets exploratoires, et surveillez les release notes des compilateurs pour suivre la progression du support.
