# 36.3.1 — Syntaxe Python-like

## Section 36.3 : fmt — Formatage avancé (pré-C++23)

---

## Introduction

La syntaxe de formatage de {fmt} est directement inspirée du [PEP 3101](https://peps.python.org/pep-3101/) de Python, et c'est cette même syntaxe qui a été adoptée par `std::format` (C++20) puis `std::print` (C++23). Si vous avez lu les **sections 2.7 et 12.7**, vous connaissez déjà les bases. Cette sous-section se concentre sur les aspects avancés et les fonctionnalités spécifiques à {fmt} qui n'ont pas (encore) été standardisées — arguments nommés, formatage de conteneurs, formateurs personnalisés et optimisations de performance.

---

## Rappel : le mini-langage de formatage

La syntaxe de base est partagée entre {fmt}, `std::format` et `std::print`. Les placeholders `{}` sont remplacés par les arguments dans l'ordre :

```cpp
#include <fmt/core.h>

fmt::print("Hello, {}!\n", "world");           // Hello, world!  
fmt::print("{} + {} = {}\n", 2, 3, 5);         // 2 + 3 = 5  
```

Le mini-langage de formatage à l'intérieur des accolades suit la grammaire :

```
{[index_ou_nom]:[remplissage][alignement][signe][#][0][largeur][.précision][type]}
```

Un récapitulatif rapide des spécificateurs les plus utilisés dans un contexte CLI :

```cpp
// Alignement et largeur — essentiel pour les tableaux
fmt::print("{:<20} {:>10} {:^8}\n", "NOM", "TAILLE", "STATUS");  
fmt::print("{:<20} {:>10} {:^8}\n", "config.yaml", "2.4 KB", "OK");  
fmt::print("{:<20} {:>10} {:^8}\n", "database.sql", "156 MB", "WARN");  

// Résultat :
// NOM                    TAILLE  STATUS
// config.yaml            2.4 KB    OK
// database.sql           156 MB   WARN

// Remplissage personnalisé
fmt::print("{:=^40}\n", " Rapport ");
// =============== Rapport ================

fmt::print("{:─<40}\n", "Section 1 ");
// Section 1 ──────────────────────────────

// Nombres
fmt::print("Entier     : {:d}\n", 42);         // 42  
fmt::print("Hex        : {:#x}\n", 255);        // 0xff  
fmt::print("Hex maj    : {:#X}\n", 255);        // 0XFF  
fmt::print("Octal      : {:#o}\n", 255);        // 0377  
fmt::print("Binaire    : {:#b}\n", 42);          // 0b101010  
fmt::print("Flottant   : {:.2f}\n", 3.14159);   // 3.14  
fmt::print("Scientif   : {:.3e}\n", 0.00042);   // 4.200e-04  
fmt::print("Général    : {:.4g}\n", 3.14159);   // 3.142  
fmt::print("Séparateur : {:L}\n", 1234567);      // 1,234,567 (locale)  

// Padding numérique avec zéros
fmt::print("ID : {:06d}\n", 42);                // ID : 000042  
fmt::print("Addr : {:#010x}\n", 0xDEAD);        // Addr : 0x0000dead  

// Signe
fmt::print("{:+d} {:+d}\n", 42, -42);           // +42 -42  
fmt::print("{: d} {: d}\n", 42, -42);           //  42 -42  

// Booléens
fmt::print("Actif : {}\n", true);               // Actif : true
```

### Index de position

Par défaut, les arguments sont consommés dans l'ordre. Les index explicites permettent de réordonner ou de réutiliser un argument :

```cpp
fmt::print("{0} a {1} ans. {0} habite à {2}.\n",
           "Alice", 30, "Paris");
// Alice a 30 ans. Alice habite à Paris.

// Utile pour l'internationalisation :
// En français : "{0} a transféré {1} à {2}"
// En japonais : "{0}は{2}に{1}を転送しました"
```

> ⚠️ **Contrainte** : on ne peut pas mélanger index explicites et implicites dans la même format string. C'est soit `{} {} {}`, soit `{0} {1} {2}`, jamais un mélange des deux.

## Arguments nommés (exclusif {fmt})

C'est une fonctionnalité que `std::format` n'offre pas. {fmt} permet de référencer les arguments par nom plutôt que par position, via `fmt::arg` :

```cpp
#include <fmt/core.h>

fmt::print("Connexion à {host}:{port} (timeout: {timeout}ms)\n",
           fmt::arg("host", "db.example.com"),
           fmt::arg("port", 5432),
           fmt::arg("timeout", 3000));
// Connexion à db.example.com:5432 (timeout: 3000ms)
```

L'intérêt est double. La format string est plus lisible — on voit immédiatement ce que chaque placeholder représente, sans compter les positions. Et quand la format string est longue ou réutilisée, le risque d'erreur de position disparaît.

### Avec `using namespace fmt::literals`

{fmt} offre un raccourci syntaxique via le suffixe `_a` :

```cpp
using namespace fmt::literals;

fmt::print("Déploiement sur {env} ({replicas} réplicas)\n",
           "env"_a = "production",
           "replicas"_a = 3);
// Déploiement sur production (3 réplicas)
```

Cette syntaxe est plus concise mais pollue le namespace. À réserver aux fichiers d'implémentation, pas aux headers.

### Cas d'usage CLI : messages de diagnostic

Les arguments nommés brillent dans les messages complexes où la lisibilité de la format string est critique :

```cpp
void report_build_result(const BuildResult& r) {
    fmt::print(
        "Build {status} en {duration:.1f}s\n"
        "  Cible     : {target}\n"
        "  Fichiers  : {compiled}/{total} compilés\n"
        "  Warnings  : {warnings}\n"
        "  Erreurs   : {errors}\n",
        fmt::arg("status", r.success ? "réussi" : "ÉCHOUÉ"),
        fmt::arg("duration", r.elapsed_seconds),
        fmt::arg("target", r.target_name),
        fmt::arg("compiled", r.files_compiled),
        fmt::arg("total", r.files_total),
        fmt::arg("warnings", r.warning_count),
        fmt::arg("errors", r.error_count)
    );
}
```

Sans arguments nommés, la même format string avec sept `{}` positionnels serait beaucoup plus difficile à relire et à maintenir.

## Formatage de conteneurs et ranges (exclusif {fmt})

L'inclusion de `<fmt/ranges.h>` rend formattables tous les conteneurs standard et les types compatibles avec les ranges. C'est un gain de productivité considérable pour le diagnostic et le débogage :

```cpp
#include <fmt/ranges.h>
#include <vector>
#include <map>
#include <set>
#include <tuple>

std::vector<std::string> services = {"nginx", "postgres", "redis"};  
fmt::print("Services : {}\n", services);  
// Services : ["nginx", "postgres", "redis"]

std::set<int> ports = {80, 443, 8080};  
fmt::print("Ports ouverts : {}\n", ports);  
// Ports ouverts : {80, 443, 8080}

std::map<std::string, int> metrics = {{"ok", 142}, {"warn", 7}, {"err", 2}};  
fmt::print("Métriques : {}\n", metrics);  
// Métriques : {"err": 2, "ok": 142, "warn": 7}

auto info = std::make_tuple("web-01", "192.168.1.10", 8080);  
fmt::print("Nœud : {}\n", info);  
// Nœud : ("web-01", "192.168.1.10", 8080)
```

### Formatage personnalisé de conteneurs avec `fmt::join`

`fmt::join` permet de contrôler le séparateur et le formatage individuel des éléments :

```cpp
std::vector<double> latencies = {1.2, 3.7, 0.8, 2.1, 5.3};  
fmt::print("Latences : {:.1f}\n", fmt::join(latencies, ", "));  
// Latences : 1.2, 3.7, 0.8, 2.1, 5.3

std::vector<std::string> args = {"--verbose", "--format", "json"};  
fmt::print("Commande : myapp {}\n", fmt::join(args, " "));  
// Commande : myapp --verbose --format json

// Séparateur avec retour à la ligne pour une liste verticale
std::vector<std::string> files = {"main.cpp", "utils.cpp", "config.hpp"};  
fmt::print("Fichiers modifiés :\n  • {}\n", fmt::join(files, "\n  • "));  
// Fichiers modifiés :
//   • main.cpp
//   • utils.cpp
//   • config.hpp
```

`fmt::join` est particulièrement utile pour construire des sorties CLI lisibles à partir de données structurées, sans écrire de boucle manuelle.

## Formatage de dates et durées

`<fmt/chrono.h>` offre un formatage riche des types `std::chrono`, avec la syntaxe `strftime` familière :

```cpp
#include <fmt/chrono.h>
#include <chrono>

// Timestamp courant
auto now = std::chrono::system_clock::now();  
fmt::print("[{:%Y-%m-%d %H:%M:%S}] Démarrage du service\n", now);  
// [2026-03-17 14:32:05] Démarrage du service

// Format court
fmt::print("Date : {:%d/%m/%Y}\n", now);
// Date : 17/03/2026

// Heure seule
fmt::print("Heure : {:%H:%M}\n", now);
// Heure : 14:32

// Durées
auto elapsed = std::chrono::milliseconds(3742);  
fmt::print("Durée : {}\n", elapsed);  
// Durée : 3742ms

auto build_time = std::chrono::duration<double>(127.356);  
fmt::print("Build : {:.1f}s\n",  
           std::chrono::duration<double>(build_time).count());

// Durée formatée en composants
auto uptime = std::chrono::hours(26) + std::chrono::minutes(13);  
fmt::print("Uptime : {:%H heures %M minutes}\n", uptime);  
// Uptime : 26 heures 13 minutes
```

Dans un outil CLI, le formatage de durées est omniprésent : temps de compilation, latences réseau, uptime de services, durée d'exécution de commandes. Avoir un formatage intégré évite les conversions manuelles `seconds / 3600` et les `std::setw` pénibles de `<iomanip>`.

## Formateurs personnalisés (`fmt::formatter`)

Pour afficher vos propres types avec `fmt::print`, vous spécialisez le template `fmt::formatter<T>`. C'est le même mécanisme que pour `std::format`, avec une syntaxe identique :

```cpp
#include <fmt/format.h>
#include <string>

struct ServiceStatus {
    std::string name;
    int port;
    bool running;
    double uptime_hours;
};

template <>  
struct fmt::formatter<ServiceStatus> {  
    // Stocker le format de présentation
    char presentation = 's';  // 's' = short, 'v' = verbose

    constexpr auto parse(format_parse_context& ctx) {
        auto it = ctx.begin(), end = ctx.end();
        if (it != end && (*it == 's' || *it == 'v')) {
            presentation = *it++;
        }
        if (it != end && *it != '}') {
            throw fmt::format_error("format invalide pour ServiceStatus");
        }
        return it;
    }

    auto format(const ServiceStatus& s, format_context& ctx) const {
        if (presentation == 'v') {
            return fmt::format_to(ctx.out(),
                "Service '{}' sur port {} — {} (uptime: {:.1f}h)",
                s.name, s.port,
                s.running ? "ACTIF" : "ARRÊTÉ",
                s.uptime_hours);
        }
        // Mode court par défaut
        return fmt::format_to(ctx.out(), "{}:{} [{}]",
                              s.name, s.port,
                              s.running ? "UP" : "DOWN");
    }
};
```

Utilisation :

```cpp
ServiceStatus svc{"nginx", 8080, true, 72.5};

fmt::print("{}\n", svc);
// nginx:8080 [UP]

fmt::print("{:v}\n", svc);
// Service 'nginx' sur port 8080 — ACTIF (uptime: 72.5h)

// Fonctionne aussi dans les conteneurs
std::vector<ServiceStatus> services = {
    {"nginx", 80, true, 168.2},
    {"api", 3000, true, 24.1},
    {"worker", 0, false, 0.0}
};
for (const auto& s : services) {
    fmt::print("  {:v}\n", s);
}
// Service 'nginx' sur port 80 — ACTIF (uptime: 168.2h)
// Service 'api' sur port 3000 — ACTIF (uptime: 24.1h)
// Service 'worker' sur port 0 — ARRÊTÉ (uptime: 0.0h)
```

### Points clés du formateur personnalisé

La méthode `parse()` analyse le spécificateur de format (ce qui vient après `:` dans `{:v}`). Elle permet de supporter plusieurs modes de présentation pour le même type. Si votre type n'a pas besoin de spécificateur, la version minimale est :

```cpp
template <>  
struct fmt::formatter<MonType> : fmt::formatter<std::string_view> {  
    auto format(const MonType& val, format_context& ctx) const {
        return fmt::format_to(ctx.out(), "...", val.member);
    }
};
```

Hériter de `fmt::formatter<std::string_view>` fournit un `parse()` par défaut qui accepte les spécificateurs d'alignement standard — votre type supporte alors `{:<20}`, `{:>30}`, etc., gratuitement.

### Compatibilité avec std::format

Un formateur écrit pour {fmt} est compatible avec `std::format` si vous remplacez `fmt::formatter` par `std::formatter` et `fmt::format_to` par `std::format_to`. La structure est identique. En pratique, beaucoup de projets définissent le formateur pour {fmt} et ajoutent un alias pour le standard :

```cpp
// Formateur {fmt}
template <>  
struct fmt::formatter<ServiceStatus> { /* ... */ };  

// Alias pour std::format (C++20+)
template <>  
struct std::formatter<ServiceStatus> : fmt::formatter<ServiceStatus> {};  
```

## `fmt::format_to` et `fmt::memory_buffer`

Pour les scénarios de performance — construire une grande sortie en mémoire avant de l'écrire en une seule fois — {fmt} offre `format_to` et `memory_buffer` :

```cpp
#include <fmt/format.h>

// memory_buffer : buffer stack-allocated avec croissance dynamique
fmt::memory_buffer buf;

// Construire un tableau en mémoire
fmt::format_to(std::back_inserter(buf),
               "{:<30} {:>10} {:>10}\n", "FICHIER", "LIGNES", "TAILLE");
fmt::format_to(std::back_inserter(buf),
               "{:─<52}\n", "");

for (const auto& f : files) {
    fmt::format_to(std::back_inserter(buf),
                   "{:<30} {:>10} {:>10}\n",
                   f.name, f.lines, f.size_str());
}

// Écrire tout d'un coup (une seule opération I/O)
fmt::print("{}", fmt::to_string(buf));
```

Pourquoi ne pas simplement faire `fmt::print` à chaque ligne ? Pour deux raisons :

- **Performance I/O** : chaque appel à `print` est un appel système `write`. Accumuler dans un buffer et écrire en une fois réduit le nombre de syscalls — une différence mesurable quand on génère des milliers de lignes.  
- **Atomicité visuelle** : en multi-thread ou quand la sortie est partagée, écrire en un bloc évite l'entrelacement de lignes provenant de sources différentes.

### `fmt::format_to` vers un `std::string`

Si vous n'avez pas besoin de l'optimisation stack du `memory_buffer`, `format_to` fonctionne aussi avec un `std::string` :

```cpp
std::string result;  
fmt::format_to(std::back_inserter(result),  
               "Total : {} fichiers, {} lignes\n", count, total_lines);
```

## Différences résiduelles entre {fmt} et std::format

En 2026, avec GCC 15 et Clang 20, `std::format` et `std::print` sont pleinement fonctionnels. Voici ce qui reste spécifique à {fmt} :

| Fonctionnalité | {fmt} | std::format / std::print |
|----------------|:-----:|:------------------------:|
| Syntaxe de base (`{}`, alignement, types) | ✅ | ✅ |
| `print` vers `stdout` | ✅ | ✅ (C++23) |
| `print` vers `FILE*` (stderr, fichier) | ✅ | ❌ (`std::print` accepte `ostream`) |
| Couleurs et styles | ✅ | ❌ |
| Arguments nommés (`fmt::arg`) | ✅ | ❌ |
| Formatage de conteneurs (`fmt/ranges.h`) | ✅ | ✅ (C++23, partiel) |
| `fmt::join` | ✅ | ❌ |
| `memory_buffer` | ✅ | ❌ |
| Formatage `chrono` | ✅ (étendu) | ✅ (basique, C++20) |
| Formateurs personnalisés | ✅ | ✅ (syntaxe identique) |
| Performances de compilation | ✅✅ | ✅ |

En résumé : la syntaxe de formatage est la même partout, mais {fmt} offre des outils supplémentaires autour du formatage — couleurs, arguments nommés, `join`, buffers — qui sont spécifiquement utiles dans les outils CLI.

## Bonnes pratiques pour les outils CLI

### Utiliser fmt::print pour les sorties, std::format pour la construction

Un pattern clair : {fmt} pour écrire directement sur les flux, `std::format` pour construire des chaînes intermédiaires.

```cpp
// Construction d'une chaîne (std::format suffit)
auto label = std::format("[{}/{}]", current, total);

// Écriture sur stderr avec couleur (besoin de {fmt})
fmt::print(stderr, fg(fmt::color::green), "{} Traitement terminé\n", label);
```

### Préférer `fmt::print(stderr, ...)` à `fmt::print(fmt::fg(...), ...)`

Pour les messages de diagnostic, écrivez toujours sur `stderr`. Cela permet à l'utilisateur de rediriger `stdout` sans perdre les messages d'état :

```cpp
// ✓ Bon : diagnostic sur stderr
fmt::print(stderr, fg(fmt::color::yellow), "⚠ Fichier volumineux : {}\n", path);

// ✗ Risqué : diagnostic sur stdout, pollue les pipes
fmt::print(fg(fmt::color::yellow), "⚠ Fichier volumineux : {}\n", path);
```

### Construire des tableaux alignés

Les outils CLI affichent souvent des données tabulaires. Voici un pattern réutilisable :

```cpp
void print_table(const std::vector<std::vector<std::string>>& rows,
                 const std::vector<int>& widths,
                 const std::vector<char>& aligns) {
    for (const auto& row : rows) {
        for (size_t i = 0; i < row.size(); ++i) {
            char align = i < aligns.size() ? aligns[i] : '<';
            int width = i < widths.size() ? widths[i] : 10;

            switch (align) {
                case '<': fmt::print("{:<{}} ", row[i], width); break;
                case '>': fmt::print("{:>{}} ", row[i], width); break;
                case '^': fmt::print("{:^{}} ", row[i], width); break;
            }
        }
        fmt::print("\n");
    }
}
```

Notez l'utilisation de `{:<{}}` — la largeur elle-même est un argument dynamique. C'est une fonctionnalité du mini-langage : tout paramètre numérique peut être remplacé par `{}` pour être fourni dynamiquement.

```cpp
int width = 20;  
fmt::print("{:<{}}\n", "hello", width);  
// hello               (20 caractères de large)

int precision = 3;  
fmt::print("{:.{}f}\n", 3.14159, precision);  
// 3.142
```
