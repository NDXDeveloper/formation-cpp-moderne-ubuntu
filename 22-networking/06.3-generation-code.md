# 22.6.3 — Génération de code et implémentation

## Section 22.6 : gRPC et Protocol Buffers

---

## Introduction

Le fichier `.proto` est écrit, la toolchain est configurée. Cette section franchit l'étape suivante : transformer la définition d'interface en **code C++ fonctionnel** — un serveur qui implémente les méthodes RPC et un client qui les appelle comme des fonctions locales.

Nous partirons d'un service concret, examinerons le code généré par `protoc`, puis construirons un serveur et un client complets dans les deux styles proposés par gRPC C++ : l'API **synchrone** (simple, un thread par appel) et l'API **asynchrone basée sur les callbacks** (performante, adaptée à la production).

---

## Le fichier `.proto` de référence

Tous les exemples de cette section s'appuient sur ce service :

```protobuf
// proto/taskmanager.proto
syntax = "proto3";

package taskmanager.v1;

import "google/protobuf/timestamp.proto";  
import "google/protobuf/empty.proto";  

// Namespace C++ généré : taskmanager::v1

service TaskManager {
    // Unaire
    rpc CreateTask (CreateTaskRequest) returns (CreateTaskResponse);
    rpc GetTask (GetTaskRequest) returns (GetTaskResponse);
    rpc ListTasks (ListTasksRequest) returns (ListTasksResponse);
    
    // Streaming serveur
    rpc WatchTasks (WatchTasksRequest) returns (stream TaskEvent);
}

message Task {
    string id = 1;
    string title = 2;
    string description = 3;
    bool completed = 4;
    google.protobuf.Timestamp created_at = 5;
}

message CreateTaskRequest {
    string title = 1;
    string description = 2;
}

message CreateTaskResponse {
    Task task = 1;
}

message GetTaskRequest {
    string task_id = 1;
}

message GetTaskResponse {
    Task task = 1;
}

message ListTasksRequest {
    int32 page_size = 1;
    string page_token = 2;
}

message ListTasksResponse {
    repeated Task tasks = 1;
    string next_page_token = 2;
}

message WatchTasksRequest {}

message TaskEvent {
    enum EventType {
        EVENT_TYPE_UNSPECIFIED = 0;
        EVENT_TYPE_CREATED = 1;
        EVENT_TYPE_UPDATED = 2;
        EVENT_TYPE_DELETED = 3;
    }
    EventType type = 1;
    Task task = 2;
}
```

## Ce que `protoc` génère

La commande de génération (exécutée automatiquement par CMake, voir section 22.6.1) produit quatre fichiers :

```bash
protoc --proto_path=proto \
       --cpp_out=build/generated \
       --grpc_out=build/generated \
       --plugin=protoc-gen-grpc=$(which grpc_cpp_plugin) \
       proto/taskmanager.proto
```

```
build/generated/
├── taskmanager.pb.h          # Classes de messages Protobuf
├── taskmanager.pb.cc         # Implémentation sérialisation/désérialisation
├── taskmanager.grpc.pb.h     # Stubs gRPC (client + serveur)
└── taskmanager.grpc.pb.cc    # Implémentation transport gRPC
```

### Les classes de messages (`taskmanager.pb.h`)

Pour chaque `message` du `.proto`, `protoc` génère une classe C++ complète :

```cpp
// Simplifié pour la lisibilité — le code réel est plus verbeux

namespace taskmanager::v1 {

class Task : public google::protobuf::Message {  
public:  
    // Constructeurs
    Task();
    Task(const Task& other);
    Task(Task&& other) noexcept;
    Task& operator=(const Task& other);
    Task& operator=(Task&& other) noexcept;

    // Accesseurs pour chaque champ
    const std::string& id() const;
    void set_id(const std::string& value);
    void set_id(std::string&& value);
    std::string* mutable_id();
    void clear_id();

    const std::string& title() const;
    void set_title(const std::string& value);
    // ... idem pour chaque champ string

    bool completed() const;
    void set_completed(bool value);
    void clear_completed();

    // Champ message imbriqué
    bool has_created_at() const;
    const google::protobuf::Timestamp& created_at() const;
    google::protobuf::Timestamp* mutable_created_at();
    void clear_created_at();

    // Sérialisation
    bool SerializeToString(std::string* output) const;
    bool ParseFromString(const std::string& data);
    size_t ByteSizeLong() const;

    // Debug
    std::string DebugString() const;
    std::string ShortDebugString() const;
};

} // namespace taskmanager::v1
```

Le pattern d'accesseurs est uniforme : `field()` pour la lecture, `set_field()` pour l'écriture de scalaires, `mutable_field()` pour obtenir un pointeur modifiable sur un sous-message ou un string, `has_field()` pour les champs optionnels et les sous-messages, et `clear_field()` pour réinitialiser à la valeur par défaut.

### Les stubs gRPC (`taskmanager.grpc.pb.h`)

Le fichier gRPC généré contient trois éléments clés :

```cpp
namespace taskmanager::v1 {

class TaskManager final {  
public:  
    // ── 1. Classe de base du SERVICE (serveur) ──
    class Service : public grpc::Service {
    public:
        // Méthodes virtuelles à implémenter
        virtual grpc::Status CreateTask(
            grpc::ServerContext* context,
            const CreateTaskRequest* request,
            CreateTaskResponse* response);

        virtual grpc::Status GetTask(
            grpc::ServerContext* context,
            const GetTaskRequest* request,
            GetTaskResponse* response);

        virtual grpc::Status ListTasks(
            grpc::ServerContext* context,
            const ListTasksRequest* request,
            ListTasksResponse* response);

        // Streaming serveur : retourne un ServerWriter
        virtual grpc::Status WatchTasks(
            grpc::ServerContext* context,
            const WatchTasksRequest* request,
            grpc::ServerWriter<TaskEvent>* writer);
    };

    // ── 2. STUB client (synchrone) ──
    class Stub {
    public:
        explicit Stub(std::shared_ptr<grpc::Channel> channel);

        grpc::Status CreateTask(
            grpc::ClientContext* context,
            const CreateTaskRequest& request,
            CreateTaskResponse* response);

        grpc::Status GetTask(
            grpc::ClientContext* context,
            const GetTaskRequest& request,
            GetTaskResponse* response);

        grpc::Status ListTasks(
            grpc::ClientContext* context,
            const ListTasksRequest& request,
            ListTasksResponse* response);

        // Streaming serveur : retourne un ClientReader
        std::unique_ptr<grpc::ClientReader<TaskEvent>> WatchTasks(
            grpc::ClientContext* context,
            const WatchTasksRequest& request);
    };

    // Factory pour créer un stub
    static std::unique_ptr<Stub> NewStub(
        std::shared_ptr<grpc::Channel> channel);
};

} // namespace taskmanager::v1
```

L'architecture est symétrique : le serveur hérite de `TaskManager::Service` et implémente les méthodes virtuelles, le client utilise `TaskManager::Stub` et appelle les méthodes correspondantes. Les signatures sont identiques — mêmes types de requête et de réponse — mais les rôles sont inversés.

## Implémentation du serveur synchrone

### La classe d'implémentation

Le serveur hérite de la classe `Service` générée et implémente chaque méthode RPC :

```cpp
// src/server.cpp
#include <grpcpp/grpcpp.h>
#include "taskmanager.grpc.pb.h"

#include <print>
#include <map>
#include <mutex>
#include <string>
#include <chrono>
#include <atomic>

using namespace taskmanager::v1;

class TaskManagerImpl final : public TaskManager::Service {  
public:  
    grpc::Status CreateTask(
        grpc::ServerContext* context,
        const CreateTaskRequest* request,
        CreateTaskResponse* response) override
    {
        // Générer un ID unique
        std::string id = "task-" + std::to_string(next_id_++);

        // Créer la tâche
        Task task;
        task.set_id(id);
        task.set_title(request->title());
        task.set_description(request->description());
        task.set_completed(false);

        // Timestamp de création
        auto now = std::chrono::system_clock::now();
        auto seconds = std::chrono::duration_cast<std::chrono::seconds>(
            now.time_since_epoch()).count();
        task.mutable_created_at()->set_seconds(seconds);

        // Stocker
        {
            std::lock_guard lock{mutex_};
            tasks_[id] = task;
        }

        // Remplir la réponse
        *response->mutable_task() = task;

        std::println("[CreateTask] Créé: {} - '{}'", id, task.title());
        return grpc::Status::OK;
    }

    grpc::Status GetTask(
        grpc::ServerContext* context,
        const GetTaskRequest* request,
        GetTaskResponse* response) override
    {
        std::lock_guard lock{mutex_};
        auto it = tasks_.find(request->task_id());

        if (it == tasks_.end()) {
            return grpc::Status(
                grpc::StatusCode::NOT_FOUND,
                "Tâche non trouvée: " + request->task_id()
            );
        }

        *response->mutable_task() = it->second;
        return grpc::Status::OK;
    }

    grpc::Status ListTasks(
        grpc::ServerContext* context,
        const ListTasksRequest* request,
        ListTasksResponse* response) override
    {
        std::lock_guard lock{mutex_};

        int page_size = request->page_size() > 0 ? request->page_size() : 10;
        bool past_token = request->page_token().empty();
        int count = 0;

        for (const auto& [id, task] : tasks_) {
            if (!past_token) {
                if (id == request->page_token()) {
                    past_token = true;
                }
                continue;
            }

            if (count >= page_size) {
                response->set_next_page_token(id);
                break;
            }

            *response->add_tasks() = task;
            count++;
        }

        return grpc::Status::OK;
    }

    grpc::Status WatchTasks(
        grpc::ServerContext* context,
        const WatchTasksRequest* request,
        grpc::ServerWriter<TaskEvent>* writer) override
    {
        std::println("[WatchTasks] Client connecté au stream");

        // Envoyer les tâches existantes comme événements initiaux
        {
            std::lock_guard lock{mutex_};
            for (const auto& [id, task] : tasks_) {
                TaskEvent event;
                event.set_type(TaskEvent::EVENT_TYPE_CREATED);
                *event.mutable_task() = task;

                if (!writer->Write(event)) {
                    // Le client a fermé le stream
                    return grpc::Status::OK;
                }
            }
        }

        // Garder le stream ouvert (simplifié — en production,
        // utiliser une queue d'événements avec notification)
        while (!context->IsCancelled()) {
            std::this_thread::sleep_for(std::chrono::seconds(1));
        }

        std::println("[WatchTasks] Client déconnecté");
        return grpc::Status::OK;
    }

private:
    std::map<std::string, Task> tasks_;
    std::mutex mutex_;
    std::atomic<int> next_id_{1};
};
```

### Analyse des patterns

**`grpc::Status`** — Chaque méthode retourne un `grpc::Status`. `grpc::Status::OK` indique le succès. Pour les erreurs, construisez un `Status` avec le code approprié et un message descriptif :

```cpp
return grpc::Status(grpc::StatusCode::NOT_FOUND, "Task not found: " + id);  
return grpc::Status(grpc::StatusCode::INVALID_ARGUMENT, "Title is required");  
return grpc::Status(grpc::StatusCode::INTERNAL, "Database error");  
```

**`grpc::ServerContext`** — Le contexte contient les métadonnées de l'appel : deadline, métadonnées client, informations d'authentification, et le flag d'annulation. `context->IsCancelled()` est essentiel pour les streams : il permet de détecter quand le client se déconnecte.

**Remplissage des réponses** — Pour les champs scalaires, utilisez `set_field()`. Pour les sous-messages, `mutable_field()` retourne un pointeur que vous pouvez remplir. Pour copier un message entier, l'affectation via `*response->mutable_task() = task` est le pattern standard.

**Thread-safety** — L'API synchrone de gRPC utilise un pool de threads interne. Plusieurs appels RPC s'exécutent simultanément sur des threads différents. Si votre service a un état partagé (la map `tasks_` ici), il **doit** être protégé par un mutex ou utiliser des structures thread-safe.

### Démarrer le serveur

```cpp
void run_server(uint16_t port) {
    std::string address = "0.0.0.0:" + std::to_string(port);
    TaskManagerImpl service;

    grpc::ServerBuilder builder;

    // Écouter sur l'adresse avec des credentials insecure (pas de TLS)
    builder.AddListeningPort(address, grpc::InsecureServerCredentials());

    // Enregistrer le service
    builder.RegisterService(&service);

    // Options de performance
    builder.SetMaxReceiveMessageSize(16 * 1024 * 1024);  // 16 MiB max
    builder.SetMaxSendMessageSize(16 * 1024 * 1024);

    // Construire et démarrer
    std::unique_ptr<grpc::Server> server = builder.BuildAndStart();
    std::println("Serveur gRPC en écoute sur {}", address);

    // Bloquer jusqu'à l'arrêt
    server->Wait();
}

int main() {
    run_server(50051);
    return 0;
}
```

### Le `ServerBuilder`

`grpc::ServerBuilder` est le point de configuration central du serveur :


## Implémentation du client synchrone

### Création du stub

Le client crée un **channel** (connexion TCP vers le serveur) et un **stub** (proxy local qui traduit les appels de méthode en requêtes réseau) :

```cpp
// src/client.cpp
#include <grpcpp/grpcpp.h>
#include "taskmanager.grpc.pb.h"
#include <print>
#include <memory>

using namespace taskmanager::v1;

class TaskManagerClient {  
public:  
    explicit TaskManagerClient(std::shared_ptr<grpc::Channel> channel)
        : stub_(TaskManager::NewStub(channel)) {}

    // ── Créer une tâche ──
    std::optional<Task> CreateTask(const std::string& title,
                                    const std::string& description) {
        CreateTaskRequest request;
        request.set_title(title);
        request.set_description(description);

        CreateTaskResponse response;
        grpc::ClientContext context;
        set_deadline(context, 5);  // Timeout 5 secondes

        grpc::Status status = stub_->CreateTask(&context, request, &response);

        if (!status.ok()) {
            std::println(stderr, "CreateTask failed: {} ({})",
                         status.error_message(),
                         static_cast<int>(status.error_code()));
            return std::nullopt;
        }

        return response.task();
    }

    // ── Récupérer une tâche ──
    std::optional<Task> GetTask(const std::string& task_id) {
        GetTaskRequest request;
        request.set_task_id(task_id);

        GetTaskResponse response;
        grpc::ClientContext context;
        set_deadline(context, 5);

        grpc::Status status = stub_->GetTask(&context, request, &response);

        if (!status.ok()) {
            if (status.error_code() == grpc::StatusCode::NOT_FOUND) {
                std::println("Tâche non trouvée: {}", task_id);
            } else {
                std::println(stderr, "GetTask failed: {}", status.error_message());
            }
            return std::nullopt;
        }

        return response.task();
    }

    // ── Lister les tâches ──
    std::vector<Task> ListTasks(int page_size = 10) {
        ListTasksRequest request;
        request.set_page_size(page_size);

        ListTasksResponse response;
        grpc::ClientContext context;
        set_deadline(context, 10);

        grpc::Status status = stub_->ListTasks(&context, request, &response);

        if (!status.ok()) {
            std::println(stderr, "ListTasks failed: {}", status.error_message());
            return {};
        }

        return {response.tasks().begin(), response.tasks().end()};
    }

    // ── Streaming : observer les tâches ──
    void WatchTasks() {
        WatchTasksRequest request;
        grpc::ClientContext context;

        // Le stream n'a pas de deadline — il tourne indéfiniment
        auto reader = stub_->WatchTasks(&context, request);

        TaskEvent event;
        while (reader->Read(&event)) {
            std::println("[Event] type={} task='{}' (id={})",
                         TaskEvent::EventType_Name(event.type()),
                         event.task().title(),
                         event.task().id());
        }

        grpc::Status status = reader->Finish();
        if (!status.ok() &&
            status.error_code() != grpc::StatusCode::CANCELLED) {
            std::println(stderr, "WatchTasks ended: {}", status.error_message());
        }
    }

private:
    void set_deadline(grpc::ClientContext& context, int seconds) {
        context.set_deadline(
            std::chrono::system_clock::now() + std::chrono::seconds(seconds));
    }

    std::unique_ptr<TaskManager::Stub> stub_;
};
```

### Utilisation du client

```cpp
int main() {
    // Créer le channel (connexion au serveur)
    auto channel = grpc::CreateChannel(
        "localhost:50051",
        grpc::InsecureChannelCredentials()
    );

    TaskManagerClient client(channel);

    // Créer des tâches
    auto task1 = client.CreateTask("Installer gRPC", "Configurer la toolchain");
    auto task2 = client.CreateTask("Écrire le serveur", "Implémenter TaskManager");
    auto task3 = client.CreateTask("Écrire le client", "Tester les appels RPC");

    if (task1) {
        std::println("Créé: {} - '{}'", task1->id(), task1->title());
    }

    // Récupérer une tâche
    if (task1) {
        auto retrieved = client.GetTask(task1->id());
        if (retrieved) {
            std::println("Récupéré: '{}' (completed={})",
                         retrieved->title(), retrieved->completed());
        }
    }

    // Tâche inexistante
    client.GetTask("task-999");

    // Lister
    auto tasks = client.ListTasks();
    std::println("\n{} tâches:", tasks.size());
    for (const auto& t : tasks) {
        std::println("  [{}] {} - '{}'",
                     t.completed() ? "x" : " ", t.id(), t.title());
    }

    return 0;
}
```

## Le channel : connexion et options

Le `grpc::Channel` représente la connexion logique au serveur. Il gère automatiquement la connexion TCP sous-jacente, les reconnexions, le load balancing, et le multiplexage HTTP/2 :

```cpp
// Connexion insecure (développement)
auto channel = grpc::CreateChannel(
    "localhost:50051",
    grpc::InsecureChannelCredentials());

// Connexion TLS (production)
grpc::SslCredentialsOptions ssl_opts;  
ssl_opts.pem_root_certs = read_file("ca.pem");      // CA certificate  
// Pour mTLS (optionnel) :
// ssl_opts.pem_private_key = read_file("client.key");
// ssl_opts.pem_cert_chain = read_file("client.pem");
auto channel = grpc::CreateChannel(
    "api.example.com:443",
    grpc::SslCredentials(ssl_opts));
```

### Options du channel

```cpp
grpc::ChannelArguments args;

// Taille max des messages
args.SetMaxReceiveMessageSize(64 * 1024 * 1024);  
args.SetMaxSendMessageSize(64 * 1024 * 1024);  

// Keepalive (détection de connexions mortes)
args.SetInt(GRPC_ARG_KEEPALIVE_TIME_MS, 30000);       // Ping toutes les 30s  
args.SetInt(GRPC_ARG_KEEPALIVE_TIMEOUT_MS, 10000);    // Timeout du ping : 10s  

// Retry automatique
args.SetServiceConfigJSON(R"({
    "methodConfig": [{
        "name": [{"service": "taskmanager.v1.TaskManager"}],
        "retryPolicy": {
            "maxAttempts": 3,
            "initialBackoff": "0.1s",
            "maxBackoff": "1s",
            "backoffMultiplier": 2.0,
            "retryableStatusCodes": ["UNAVAILABLE", "DEADLINE_EXCEEDED"]
        }
    }]
})");

auto channel = grpc::CreateCustomChannel(
    "localhost:50051",
    grpc::InsecureChannelCredentials(),
    args);
```

### États du channel

Un channel passe par plusieurs états au cours de sa vie :

```
IDLE ──► CONNECTING ──► READY ──► (utilisation normale)
  ▲                       │
  │                       ▼
  └──── TRANSIENT_FAILURE ◄──── (perte de connexion)
                │
                ▼
            SHUTDOWN
```

Vous pouvez surveiller l'état du channel :

```cpp
auto state = channel->GetState(true);  // true = tenter de se connecter  
if (state == GRPC_CHANNEL_READY) {  
    // Connexion établie
}

// Attendre un changement d'état
channel->WaitForStateChange(state,
    std::chrono::system_clock::now() + std::chrono::seconds(5));
```

## Deadlines : le timeout gRPC

Les **deadlines** sont le mécanisme de timeout de gRPC. Contrairement aux timeouts classiques (durée relative), une deadline est un **point dans le temps absolu**. Cette nuance est importante : quand un appel traverse plusieurs services (A → B → C), la deadline est propagée automatiquement. Si A fixe une deadline de 5 secondes et que l'appel à B prend 3 secondes, l'appel de B vers C n'a plus que 2 secondes — pas 5.

```cpp
// Côté client : fixer la deadline
grpc::ClientContext context;  
context.set_deadline(  
    std::chrono::system_clock::now() + std::chrono::seconds(5));

grpc::Status status = stub_->GetTask(&context, request, &response);  
if (status.error_code() == grpc::StatusCode::DEADLINE_EXCEEDED) {  
    std::println(stderr, "L'appel a dépassé la deadline de 5s");
}
```

```cpp
// Côté serveur : vérifier la deadline
grpc::Status GetTask(grpc::ServerContext* context,
                     const GetTaskRequest* request,
                     GetTaskResponse* response) override {
    // Vérifier si la deadline est déjà dépassée
    if (context->IsCancelled()) {
        return grpc::Status(grpc::StatusCode::CANCELLED,
                            "Requête annulée (deadline dépassée)");
    }

    // Obtenir le temps restant
    auto deadline = context->deadline();
    auto remaining = deadline - std::chrono::system_clock::now();
    std::println("Temps restant: {}ms",
        std::chrono::duration_cast<std::chrono::milliseconds>(remaining).count());

    // ... traitement ...
    return grpc::Status::OK;
}
```

> ⚠️ **Mettez toujours une deadline sur vos appels gRPC client.** Un appel sans deadline peut bloquer indéfiniment si le serveur ne répond pas. C'est l'équivalent des timeouts HTTP — une obligation en production.

## Métadonnées : headers gRPC

Les métadonnées gRPC sont l'équivalent des headers HTTP. Elles permettent de transmettre des informations hors du message Protobuf — authentification, correlation IDs, feature flags :

```cpp
// ── Côté client : envoyer des métadonnées ──
grpc::ClientContext context;  
context.AddMetadata("authorization", "Bearer eyJhbGciOi...");  
context.AddMetadata("x-request-id", "req-abc-123");  
context.AddMetadata("x-client-version", "2.1.0");  

grpc::Status status = stub_->GetTask(&context, request, &response);

// Lire les métadonnées de réponse (trailing metadata)
auto server_metadata = context.GetServerTrailingMetadata();  
auto it = server_metadata.find("x-processing-time-ms");  
if (it != server_metadata.end()) {  
    std::println("Temps serveur: {}ms",
                 std::string(it->second.data(), it->second.size()));
}
```

```cpp
// ── Côté serveur : lire et envoyer des métadonnées ──
```

## API asynchrone basée sur les callbacks

L'API synchrone est simple mais chaque appel RPC bloque un thread. Pour les serveurs à haut débit, gRPC C++ propose une API asynchrone basée sur les **callbacks**, où le framework gère le pool de threads et notifie votre code quand une opération est terminée.

### Serveur avec callbacks

```cpp
#include <grpcpp/grpcpp.h>
#include "taskmanager.grpc.pb.h"

using namespace taskmanager::v1;

class TaskManagerCallbackImpl final
    : public TaskManager::CallbackService {
public:
    grpc::ServerUnaryReactor* CreateTask(
        grpc::CallbackServerContext* context,
        const CreateTaskRequest* request,
        CreateTaskResponse* response) override
    {
        // Créer la tâche (même logique que le serveur synchrone)
        Task task;
        std::string id = "task-" + std::to_string(next_id_++);
        task.set_id(id);
        task.set_title(request->title());
        task.set_description(request->description());

        {
            std::lock_guard lock{mutex_};
            tasks_[id] = task;
        }

        *response->mutable_task() = task;

        // Créer un reactor qui signale la fin immédiate
        auto* reactor = context->DefaultReactor();
        reactor->Finish(grpc::Status::OK);
        return reactor;
    }

    grpc::ServerUnaryReactor* GetTask(
        grpc::CallbackServerContext* context,
        const GetTaskRequest* request,
        GetTaskResponse* response) override
    {
        auto* reactor = context->DefaultReactor();

        std::lock_guard lock{mutex_};
        auto it = tasks_.find(request->task_id());

        if (it == tasks_.end()) {
            reactor->Finish(grpc::Status(
                grpc::StatusCode::NOT_FOUND, "Not found"));
        } else {
            *response->mutable_task() = it->second;
            reactor->Finish(grpc::Status::OK);
        }

        return reactor;
    }

    // ... autres méthodes ...

private:
    std::map<std::string, Task> tasks_;
    std::mutex mutex_;
    std::atomic<int> next_id_{1};
};
```

Le modèle **reactor** est le pattern moderne de gRPC C++ pour l'async. Chaque méthode retourne un `Reactor` qui contrôle le cycle de vie de l'appel. `DefaultReactor()` est le cas simple — pour les opérations qui se terminent immédiatement. Les opérations longues créent un reactor personnalisé qui appelle `Finish()` plus tard.

### Client avec callbacks

```cpp
void create_task_async(TaskManager::Stub* stub,
                        const std::string& title) {
    // Les objets doivent survivre jusqu'au callback
    auto* context = new grpc::ClientContext();
    auto* request = new CreateTaskRequest();
    auto* response = new CreateTaskResponse();

    request->set_title(title);
    context->set_deadline(
        std::chrono::system_clock::now() + std::chrono::seconds(5));

    stub->async()->CreateTask(context, request, response,
        [context, request, response](grpc::Status status) {
            if (status.ok()) {
                std::println("Créé (async): {} - '{}'",
                    response->task().id(), response->task().title());
            } else {
                std::println(stderr, "Erreur: {}", status.error_message());
            }

            // Nettoyage
            delete context;
            delete request;
            delete response;
        });
}
```

> 💡 L'utilisation de `new`/`delete` bruts ici est volontaire pour la clarté. En production, encapsulez dans des structures RAII ou des `shared_ptr` pour éviter les fuites en cas d'exception dans le callback.

## Interceptors : middleware gRPC

Les interceptors sont l'équivalent gRPC des middlewares HTTP. Ils permettent d'ajouter du comportement transversal (logging, métriques, authentification) sans modifier le code des services :

```cpp
#include <grpcpp/support/server_interceptor.h>

class LoggingInterceptor : public grpc::experimental::Interceptor {  
public:  
    explicit LoggingInterceptor(grpc::experimental::ServerRpcInfo* info)
        : info_(info) {}

    void Intercept(
        grpc::experimental::InterceptorBatchMethods* methods) override {

        if (methods->QueryInterceptionHookPoint(
                grpc::experimental::InterceptionHookPoint::
                    POST_RECV_INITIAL_METADATA)) {
            auto start = std::chrono::steady_clock::now();
            std::println("[gRPC] Début: {}", info_->method());
        }

        if (methods->QueryInterceptionHookPoint(
                grpc::experimental::InterceptionHookPoint::
                    PRE_SEND_STATUS)) {
            std::println("[gRPC] Fin: {}", info_->method());
        }

        methods->Proceed();
    }

private:
    grpc::experimental::ServerRpcInfo* info_;
};

class LoggingInterceptorFactory
    : public grpc::experimental::ServerInterceptorFactoryInterface {
public:
    grpc::experimental::Interceptor* CreateServerInterceptor(
        grpc::experimental::ServerRpcInfo* info) override {
        return new LoggingInterceptor(info);
    }
};

// Enregistrement dans le ServerBuilder
std::vector<std::unique_ptr<
    grpc::experimental::ServerInterceptorFactoryInterface>> factories;
factories.push_back(std::make_unique<LoggingInterceptorFactory>());  
builder.experimental().SetInterceptorCreators(std::move(factories));  
```

## Arrêt propre du serveur

```cpp
#include <csignal>

std::unique_ptr<grpc::Server> g_server;

void signal_handler(int) {
    if (g_server) {
        std::println("Arrêt en cours...");

        // Deadline pour l'arrêt : 5 secondes
        auto deadline = std::chrono::system_clock::now()
                        + std::chrono::seconds(5);
        g_server->Shutdown(deadline);
    }
}

int main() {
    signal(SIGINT, signal_handler);
    signal(SIGTERM, signal_handler);

    grpc::ServerBuilder builder;
    // ... configuration ...

    g_server = builder.BuildAndStart();
    std::println("Serveur démarré sur le port 50051");

    g_server->Wait();  // Bloque jusqu'au Shutdown()

    std::println("Serveur arrêté proprement");
    return 0;
}
```

`Shutdown(deadline)` arrête d'accepter de nouvelles requêtes et attend que les requêtes en cours se terminent, jusqu'à la deadline. Passé la deadline, les requêtes restantes sont annulées. C'est l'équivalent du graceful shutdown vu en section 22.2, mais intégré au framework gRPC.

## Résumé

La génération de code gRPC transforme un fichier `.proto` en code C++ prêt à l'emploi :

- **Messages Protobuf** (`.pb.h`) — Classes C++ avec accesseurs typés, sérialisation binaire, et opérations de copie/déplacement.  
- **Service serveur** (`.grpc.pb.h`) — Classe de base abstraite à hériter. Chaque méthode virtuelle correspond à un RPC.  
- **Stub client** (`.grpc.pb.h`) — Proxy qui traduit les appels de méthode en requêtes réseau transparentes.  
- **API synchrone** — Un thread par appel, simple à écrire, adapté aux charges modérées.  
- **API callbacks** — Modèle reactor, adapté aux serveurs haute performance.  
- **Deadlines** — Timeouts absolus propagés automatiquement entre services. Toujours en définir un côté client.  
- **Métadonnées** — Headers gRPC pour l'authentification, le tracing, les informations contextuelles.  
- **Interceptors** — Middleware pour le logging, les métriques, l'authentification transversale.  
- **Health checking** — Protocole standard pour l'intégration Kubernetes et les load balancers.
