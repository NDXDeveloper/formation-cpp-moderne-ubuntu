# 13.5.1 — Span statique vs dynamique

## Chapitre 13 : Conteneurs Séquentiels — std::span

---

## Introduction

Le prototype de `std::span` comporte un second paramètre template souvent ignoré :

```cpp
template <typename T, std::size_t Extent = std::dynamic_extent>  
class span;  
```

Ce paramètre `Extent` détermine si la taille du span est connue à la **compilation** (span statique) ou seulement à l'**exécution** (span dynamique). Cette distinction a des conséquences sur la taille de l'objet en mémoire, sur les vérifications possibles à la compilation, et sur les conversions implicites entre spans.

---

## Span dynamique : taille connue à l'exécution

Quand `Extent` vaut `std::dynamic_extent` (la valeur par défaut), le span stocke la taille à l'exécution dans un membre interne. C'est la forme que nous avons utilisée dans la section 13.5 :

```cpp
#include <span>
#include <vector>
#include <print>

int main() {
    std::vector<int> v{10, 20, 30, 40, 50};

    // Span dynamique — taille connue à l'exécution
    std::span<int> s(v);
    // Équivalent explicite : std::span<int, std::dynamic_extent> s(v);

    std::println("size = {}", s.size());         // 5
    std::println("sizeof(s) = {}", sizeof(s));   // 16 (pointeur + size_t)
}
```

La structure interne d'un span dynamique est un pointeur plus un compteur :

```
std::span<int> (dynamique) :
┌──────────────────┐
│ ptr_   (8 octets)│
│ size_  (8 octets)│
└──────────────────┘
  sizeof = 16 octets
```

C'est la forme la plus flexible : elle accepte n'importe quelle taille à l'exécution et sert dans la grande majorité des cas d'usage.

---

## Span statique : taille connue à la compilation

Quand `Extent` est une valeur entière, la taille fait partie du **type** et n'a pas besoin d'être stockée en mémoire. Le compilateur la connaît statiquement :

```cpp
#include <span>
#include <array>
#include <print>

int main() {
    std::array<int, 5> arr{10, 20, 30, 40, 50};

    // Span statique — la taille 5 fait partie du type
    std::span<int, 5> s(arr);

    std::println("size = {}", s.size());         // 5 (constexpr)
    std::println("sizeof(s) = {}", sizeof(s));   // 8 (pointeur uniquement)
}
```

La structure interne ne contient qu'un pointeur — la taille est encodée dans le type :

```
std::span<int, 5> (statique) :
┌──────────────────┐
│ ptr_   (8 octets)│
└──────────────────┘
  sizeof = 8 octets
  Extent = 5 (compilé dans le type, pas en mémoire)
```

L'économie de 8 octets par span peut sembler dérisoire, mais elle s'accumule si vous stockez de nombreux spans (par exemple dans un conteneur de spans), et elle permet au compilateur de générer du code plus optimisé en éliminant les vérifications de taille à l'exécution.

---

## Déduction automatique : quand obtient-on un span statique ?

La déduction du type de span dépend de la source des données :

```cpp
#include <span>
#include <vector>
#include <array>
#include <print>

int main() {
    int tab[3] = {1, 2, 3};
    std::array<int, 4> arr{1, 2, 3, 4};
    std::vector<int> vec{1, 2, 3, 4, 5};

    // Tableau C : taille connue → span STATIQUE
    std::span s1(tab);
    // s1 est std::span<int, 3>
    std::println("s1 : sizeof={}, extent={}", sizeof(s1), s1.extent);
    // sizeof=8, extent=3

    // std::array : taille connue → span STATIQUE
    std::span s2(arr);
    // s2 est std::span<int, 4>
    std::println("s2 : sizeof={}, extent={}", sizeof(s2), s2.extent);
    // sizeof=8, extent=4

    // std::vector : taille dynamique → span DYNAMIQUE
    std::span s3(vec);
    // s3 est std::span<int, std::dynamic_extent>
    std::println("s3 : sizeof={}", sizeof(s3));
    // sizeof=16
}
```

La logique est intuitive : si la source a une taille connue à la compilation (`T[N]`, `std::array<T, N>`), CTAD produit un span statique. Si la taille n'est connue qu'à l'exécution (`std::vector`, pointeur + taille), CTAD produit un span dynamique.

| Source | Span déduit | `sizeof` (x86_64) |  
|---|---|---|  
| `int[5]` | `std::span<int, 5>` | 8 |  
| `std::array<int, 5>` | `std::span<int, 5>` | 8 |  
| `std::vector<int>` | `std::span<int>` | 16 |  
| Pointeur + taille | `std::span<int>` | 16 |

## Conversions entre spans statiques et dynamiques

Les conversions implicites suivent une règle naturelle : on peut **perdre** de l'information de taille (statique → dynamique) mais pas en **inventer** (dynamique → statique).

### Statique → dynamique : conversion implicite

Un span statique se convertit implicitement en span dynamique du même type. C'est une conversion sûre — on oublie simplement l'information de taille à la compilation :

```cpp
#include <span>
#include <array>
#include <print>

void traiter(std::span<const int> donnees) {  // attend un span dynamique
    std::println("size = {}", donnees.size());
}

int main() {
    std::array<int, 4> arr{10, 20, 30, 40};
    std::span<int, 4> s_statique(arr);

    // Conversion implicite : span<int, 4> → span<int>
    traiter(s_statique);   // ✅ OK
    traiter(arr);          // ✅ OK aussi (array → span statique → span dynamique)
}
```

C'est la raison pour laquelle les fonctions acceptent généralement un `std::span<const T>` (dynamique) : elles acceptent ainsi indifféremment des spans statiques et dynamiques.

### Dynamique → statique : impossible implicitement

La conversion inverse n'est pas implicite car le compilateur ne peut pas vérifier que la taille correspond :

```cpp
#include <span>
#include <vector>

int main() {
    std::vector<int> v{1, 2, 3, 4, 5};
    std::span<int> s_dyn(v);

    // std::span<int, 5> s_stat = s_dyn;  // ❌ Erreur de compilation
}
```

Pour effectuer cette conversion, il faut passer par les sous-vues `first` ou `last` avec un paramètre template :

```cpp
#include <span>
#include <vector>
#include <print>

int main() {
    std::vector<int> v{10, 20, 30, 40, 50};
    std::span<int> s_dyn(v);

    // first<N> et last<N> retournent un span STATIQUE
    std::span<int, 3> debut = s_dyn.first<3>();
    std::span<int, 2> fin   = s_dyn.last<2>();

    std::println("debut : sizeof={}", sizeof(debut));  // 8
    std::println("fin   : sizeof={}", sizeof(fin));    // 8

    for (auto val : debut) std::print("{} ", val);
    std::println("");  // 10 20 30

    for (auto val : fin) std::print("{} ", val);
    std::println("");  // 40 50
}
```

### Statique → statique de taille différente : interdit

Deux spans statiques de tailles différentes sont des types incompatibles :

```cpp
#include <span>

int main() {
    int tab[5] = {1, 2, 3, 4, 5};
    std::span<int, 5> s5(tab);

    // std::span<int, 3> s3 = s5;  // ❌ Erreur — types incompatibles
    std::span<int, 3> s3 = s5.first<3>();  // ✅ OK — sous-vue explicite
}
```

## `first`, `last`, `subspan` : versions statiques et dynamiques

Ces trois méthodes existent chacune en deux variantes. La version avec un paramètre template produit un span statique ; la version avec un paramètre à l'exécution produit un span dynamique :

```cpp
#include <span>
#include <vector>
#include <print>

int main() {
    std::vector<int> v{10, 20, 30, 40, 50, 60};
    std::span<int> s(v);

    // Versions STATIQUES (paramètre template) → span statique
    auto a = s.first<3>();            // std::span<int, 3>
    auto b = s.last<2>();             // std::span<int, 2>
    auto c = s.subspan<1, 3>();       // std::span<int, 3>

    std::println("sizeof(a)={}, sizeof(b)={}, sizeof(c)={}",
                  sizeof(a), sizeof(b), sizeof(c));
    // 8, 8, 8

    // Versions DYNAMIQUES (paramètre runtime) → span dynamique
    std::size_t n = 3;
    auto d = s.first(n);              // std::span<int>
    auto e = s.last(n);               // std::span<int>
    auto f = s.subspan(1, n);         // std::span<int>

    std::println("sizeof(d)={}, sizeof(e)={}, sizeof(f)={}",
                  sizeof(d), sizeof(e), sizeof(f));
    // 16, 16, 16
}
```

La version statique permet au compilateur de vérifier les bornes à la compilation quand les deux spans impliqués sont statiques :

```cpp
#include <span>
#include <array>

int main() {
    std::array<int, 5> arr{1, 2, 3, 4, 5};
    std::span<int, 5> s(arr);

    auto ok = s.first<3>();   // ✅ 3 <= 5

    // auto ko = s.first<8>();  // ❌ Erreur de compilation : 8 > 5
}
```

## `extent` : inspecter la taille statique

Le membre statique `extent` permet de savoir à la compilation si un span est statique ou dynamique :

```cpp
#include <span>
#include <vector>
#include <array>
#include <print>

template <typename T, std::size_t E>  
void info(std::span<T, E> s) {  
    if constexpr (E == std::dynamic_extent) {
        std::println("Span DYNAMIQUE : size={}", s.size());
    } else {
        std::println("Span STATIQUE  : extent={}, size={}", E, s.size());
    }
}

int main() {
    std::array<int, 4> arr{1, 2, 3, 4};
    std::vector<int> vec{1, 2, 3};

    info(std::span(arr));  // Span STATIQUE  : extent=4, size=4
    info(std::span(vec));  // Span DYNAMIQUE : size=3
}
```

## Quand utiliser un span statique ?

Le span dynamique (`std::span<T>`) est le choix par défaut dans la grande majorité des cas. Le span statique (`std::span<T, N>`) se justifie dans des situations spécifiques :

**Protocoles avec tailles fixes.** Si votre API manipule des structures de taille connue (headers réseau, registres matériel, clés cryptographiques), un span statique encode cette contrainte dans le type :

```cpp
#include <span>
#include <cstdint>
#include <print>

// La clé AES-256 fait TOUJOURS 32 octets — encodé dans le type
void chiffrer(std::span<const std::uint8_t, 32> cle,
              std::span<const std::uint8_t> donnees) {
    std::println("Clé de {} octets, {} octets à chiffrer",
                  cle.size(), donnees.size());
}

int main() {
    std::array<std::uint8_t, 32> ma_cle{};
    std::vector<std::uint8_t> message{0x01, 0x02, 0x03};

    chiffrer(ma_cle, message);
    // Clé de 32 octets, 3 octets à chiffrer

    // std::array<std::uint8_t, 16> cle_courte{};
    // chiffrer(cle_courte, message);  // ❌ Erreur de compilation : 16 ≠ 32
}
```

**Algèbre linéaire et calcul numérique.** Les vecteurs et matrices de dimensions fixes sont naturellement exprimés par des spans statiques :

```cpp
#include <span>
#include <cmath>
#include <print>

double norme_3d(std::span<const double, 3> v) {
    return std::sqrt(v[0]*v[0] + v[1]*v[1] + v[2]*v[2]);
}

double produit_scalaire_3d(std::span<const double, 3> a,
                           std::span<const double, 3> b) {
    return a[0]*b[0] + a[1]*b[1] + a[2]*b[2];
}

int main() {
    std::array<double, 3> pos{3.0, 4.0, 0.0};
    std::array<double, 3> dir{1.0, 0.0, 0.0};

    std::println("Norme    : {}", norme_3d(pos));                  // 5
    std::println("Scalaire : {}", produit_scalaire_3d(pos, dir));  // 3
}
```

**Optimisation dans du code critique.** Si vous stockez de nombreux spans ou si vous travaillez dans un contexte contraint en mémoire (embarqué), l'économie de 8 octets par span peut être pertinente.

Dans tous les autres cas, préférez le span dynamique. Il est plus flexible, accepte des sources de tailles variées, et le coût des 8 octets supplémentaires est négligeable.

## Récapitulatif

| Caractéristique | Span dynamique `span<T>` | Span statique `span<T, N>` |  
|---|---|---|  
| Taille connue à la compilation | Non | Oui |  
| `sizeof` (x86_64) | 16 octets | 8 octets |  
| `extent` | `std::dynamic_extent` | `N` |  
| Construit depuis `vector` | Oui | Non (sauf si `vector.size() == N`, non vérifié) |  
| Construit depuis `array<T, N>` | Oui | Oui (si tailles compatibles) |  
| Construit depuis `T[N]` | Oui | Oui |  
| Vérifications bornes (compile-time) | Non | Oui (`first<M>` avec `M <= N`) |  
| Conversion → dynamique | Identité | Implicite |  
| Conversion → statique | Explicite (`first<N>`, `last<N>`) | Si `N` identique |  
| Cas d'usage | Paramètres de fonctions génériques | Protocoles à tailles fixes, calcul numérique |
